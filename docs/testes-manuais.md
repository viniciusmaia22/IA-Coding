# Testes manuais

## Antes de abrir Pull Request

Executar os testes abaixo.

---

## 1. Abertura do app

- [ ] O app abre sem erro visual.
- [ ] Não há erro no console.
- [ ] Campo de nova tarefa aparece.
- [ ] Botão Adicionar aparece.

---

## 2. Adicionar tarefa

- [ ] Adicionar uma tarefa válida.
- [ ] Tarefa aparece na lista.
- [ ] Campo é limpo após adicionar.
- [ ] Campo recebe foco novamente.
- [ ] Tentar adicionar tarefa vazia.
- [ ] Alerta é exibido.
- [ ] Tentar adicionar apenas espaços.
- [ ] Alerta é exibido.

---

## 3. Editar tarefa inline

- [ ] Clicar em Editar.
- [ ] Input de edição aparece.
- [ ] Salvar com botão Salvar.
- [ ] Salvar com Enter.
- [ ] Cancelar com botão Cancelar.
- [ ] Cancelar com Escape.
- [ ] Tentar salvar texto vazio.
- [ ] Alerta é exibido.
- [ ] Tarefa concluída continua concluída após edição.

---

## 4. Excluir tarefa

- [ ] Excluir tarefa pendente.
- [ ] Excluir tarefa concluída.
- [ ] Tarefa desaparece da lista.
- [ ] Contador atualiza.

---

## 5. Concluir tarefa

- [ ] Marcar tarefa como concluída.
- [ ] Texto muda visualmente.
- [ ] Contador atualiza.
- [ ] Desmarcar tarefa concluída.
- [ ] Contador atualiza.

---

## 6. Persistência

- [ ] Adicionar tarefa e recarregar.
- [ ] Tarefa continua aparecendo.
- [ ] Editar tarefa e recarregar.
- [ ] Texto editado permanece.
- [ ] Concluir tarefa e recarregar.
- [ ] Estado concluído permanece.
- [ ] Excluir tarefa e recarregar.
- [ ] Tarefa não volta.

---

## 7. Responsividade

- [ ] Testar largura menor que 480px.
- [ ] Formulário fica usável.
- [ ] Botões não ficam espremidos.
- [ ] Texto longo quebra corretamente.
- [ ] Área de ações continua acessível.

---

## 8. Acessibilidade básica

- [ ] Navegar usando Tab.
- [ ] Foco visual aparece.
- [ ] Botões são acessíveis por teclado.
- [ ] Input de edição recebe foco.
