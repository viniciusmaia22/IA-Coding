# Roadmap — App de Tarefas

Este documento organiza a evolução planejada do projeto.

---

## Fase 1 — Fundamentos

Status: concluída.

Foco:
- HTML, CSS e JavaScript puro;
- manipulação do DOM;
- estado com array;
- renderização;
- localStorage;
- testes manuais;
- documentação inicial.

---

## Fase 2 — Fluxo profissional com Git e qualidade

Status: em andamento.

Foco:
- branches;
- commits pequenos;
- Pull Requests;
- revisão de diff;
- documentação técnica;
- acessibilidade básica;
- responsividade;
- backlog;
- roadmap;
- issues no GitHub.

---

## Fase 3 — Evolução de produto

Status: planejada.

Possíveis temas:
- filtros de tarefas;
- busca;
- ordenação;
- limpeza de tarefas concluídas;
- melhoria de estados vazios;
- componentes mais organizados;
- primeiros testes automatizados.

---

## Fase 4 — Arquitetura mais avançada

Status: futuro.

Possíveis temas:
- separação em múltiplos arquivos JavaScript;
- uso de módulos;
- introdução a build tools;
- comparação com frameworks;
- possível migração para React.

---

## Critérios para avançar para a Fase 3

- Documentação técnica criada.
- Backlog criado.
- Roadmap criado.
- Issues iniciais criadas no GitHub.
- Fluxo de PR consolidado.
- Testes manuais documentados.
